# Delta: Approval Requests

## ADDED Requirements

### Requirement: Отзыв заявки до начала согласования

Система SHALL разрешать автору с permission `approval-request.withdraw-own` отозвать собственную заявку в статусе `Submitted`, если согласование ещё не началось.

#### Scenario: Автор успешно отзывает заявку

- GIVEN заявка находится в статусе `Submitted`
- AND согласование заявки ещё не началось
- AND текущий пользователь является автором заявки
- AND у пользователя есть permission `approval-request.withdraw-own`
- WHEN пользователь отзывает заявку
- THEN система атомарно изменяет статус заявки на `Withdrawn`
- AND система возвращает обновлённые статус и version заявки
- AND система создаёт audit event успешного отзыва

#### Scenario: У пользователя нет permission для отзыва

- GIVEN заявка находится в статусе `Submitted`
- AND текущий пользователь является автором заявки
- AND у пользователя нет permission `approval-request.withdraw-own`
- WHEN пользователь пытается отозвать заявку
- THEN система отклоняет операцию с кодом `403`
- AND статус заявки не изменяется
- AND business audit event успешного отзыва не создаётся

#### Scenario: Пользователь пытается отозвать чужую заявку

- GIVEN заявка находится в статусе `Submitted`
- AND согласование заявки ещё не началось
- AND текущий пользователь не является автором заявки
- WHEN пользователь пытается отозвать заявку
- THEN система отклоняет операцию с кодом `403`
- AND статус заявки не изменяется
- AND система не раскрывает пользователю дополнительные данные заявки

#### Scenario: Согласование началось одновременно с отзывом

- GIVEN заявка находится в статусе `Submitted`
- AND текущий пользователь является автором заявки с необходимым permission
- AND процесс согласования успел зафиксировать начало до изменения статуса
- WHEN пользователь пытается отозвать заявку
- THEN система отклоняет операцию с кодом `409`
- AND возвращает error code `APPROVAL_ALREADY_STARTED`
- AND статус заявки не изменяется
- AND audit event успешного отзыва не создаётся

#### Scenario: Пользователь повторно отзывает уже отозванную заявку

- GIVEN заявка находится в статусе `Withdrawn`
- AND текущий пользователь является автором заявки с необходимым permission
- WHEN пользователь повторно отправляет запрос отзыва
- THEN система отклоняет операцию с кодом `409`
- AND возвращает error code `REQUEST_ALREADY_WITHDRAWN`
- AND повторный audit event не создаётся

### Requirement: Аудит успешного отзыва заявки

Система SHALL формировать audit event успешного отзыва заявки и обеспечивать его доставку в установленный корпоративный audit topic.

#### Scenario: Успешный отзыв создаёт audit event

- GIVEN заявка успешно перешла из `Submitted` в `Withdrawn`
- WHEN изменение заявки успешно зафиксировано
- THEN система формирует audit event успешного отзыва
- AND event type равен `APPROVAL_REQUEST_WITHDRAWN`
- AND entity type равен `approval-request`
- AND entity ID равен идентификатору отозванной заявки
- AND actor ID равен идентификатору пользователя
- AND event содержит correlation ID исходного запроса
- AND event не содержит полное содержание заявки или чувствительные данные

#### Scenario: Audit event доставляется в Kafka

- GIVEN audit event успешного отзыва сформирован
- AND для event настроен корпоративный audit topic
- WHEN система доставляет audit event
- THEN сообщение публикуется в настроенный Kafka topic
- AND сообщение соответствует audit schema version `1`
- AND сообщение содержит уникальный `eventId` для deduplication

#### Scenario: Kafka временно недоступна

- GIVEN audit event успешного отзыва сформирован
- AND Kafka временно недоступна
- WHEN первая попытка доставки сообщения завершается ошибкой
- THEN успешный отзыв заявки не откатывается
- AND система доставляет event после восстановления Kafka
- AND ошибка доставки доступна для операционного контроля

### Requirement: Видимость действия отзыва

Система SHALL показывать action отзыва в UI только при известных клиенту условиях доступности, не заменяя backend authorization.

#### Scenario: UI показывает action доступному автору

- GIVEN пользователь является автором заявки
- AND заявка имеет статус `Submitted`
- AND согласование по данным UI ещё не началось
- AND у пользователя есть permission `approval-request.withdraw-own`
- WHEN пользователь открывает заявку
- THEN UI показывает action отзыва

#### Scenario: Устаревший UI не обходит backend validation

- GIVEN UI показывает action на основании устаревших данных
- AND согласование уже началось на backend
- WHEN пользователь подтверждает отзыв
- THEN backend отклоняет операцию с кодом `409`
- AND UI обновляет данные заявки и показывает установленное сообщение
